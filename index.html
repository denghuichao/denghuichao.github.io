Java运行时内存区域和OOM

2017-07-23

Java运行时数据区域

程序计数器

  • 字节码解释器工作时通过改变这个计数器的值来选取下一条要执行的字节码指令。
  • 分支、循环、跳转、异常处理、线程恢复等基础功能的实现依赖该计数器
  • 线程私有
  • 唯一一个没有规定任何OOM的区域

Java虚拟机栈

  • 线程私有
  • 描述Java方法执行的内存模型，方法执行时创建栈帧(Stack Frame)保存局部变量表，操作数栈、动态链接、方法出口信息。
  • 局部变量表存放编译器可知的基本数据类型、对象引用和returnAddress类型
  • 进入一个方法时，栈帧大小完全确定，运行时不会改变
  • 虚拟机栈可能抛出StackOverflowError和OutOfMemoryError

重入锁和读写锁

2017-07-16

重入锁

什么是可重入？

已经获得锁的线程再次请求锁，能够连续多次获得该锁而不会被阻塞。

为什么需要重入？

子类改写了父类的synchonized方法，然后调用父类中的方法，此时如果没有重入的锁，那么这段代码将产生死锁。

如果一个递归的方法是synchonized的，那么在拥有锁的情况下调用自身将会死锁。

synchronized关键字隐式支持锁的重入。

Java中的锁

2017-07-15

锁的作用

控制多个线程访问共享资源的方式，一般来说，锁防止多个线程同时访问共享资源（共享锁例外），Java5之前用synchronized关键字实现锁的功能，Java5后新增了Lock接口及其实现，提供了显式锁的功能。

synchronized

  • 隐式获取锁
  • 锁的获取和释放固化
  • 简化同步的管理
  • 可扩展性没有显式锁好

Ubuntu 16 安装 ElasticSearch

2017-07-12

Install ES in Ubuntu

  • wget -qO - https://packages.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
  • echo “deb https://packages.elastic.co/elasticsearch/2.x/debian stable main” | sudo tee -a /etc/apt/sources.list.d/elasticsearch.list
  • sudo apt-get update
  • sudo apt-get install -y elasticsearch

start ES in Ubuntu

  • sudo systemctl daemon-reload
  • sudo systemctl enable elasticsearch.service
  • sudo systemctl start elasticsearch.service
  • curl -X GET http://localhost:9200

Install Kibana in Ubuntu

  • wget -qO - https://packages.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
  • echo “deb http://packages.elastic.co/kibana/4.5/debian stable main” | sudo tee -a /etc/apt/sources.list
  • sudo apt-get update && sudo apt-get install kibana

Start Kabana

  • sudo systemctl start kibana
  • sudo systemctl enable kibana
  • http://localhost:5601/

用Java写个爬虫IP代理池

2017-07-08

简介

最近在玩爬虫, 很多网站有反爬虫机制,会对访问频率太高的IP进行禁封.于是到网上去找免费的代理,使用过程中发现这些代理的质量不高,很多都不能使用,因此想到做个代理池.程序主动收集代理IP,并定时对这些代理进行可用性验证,同时提供接口获取可用的代理供爬虫使用.

代理IP来源

有很多网站提供代理IP服务,有的免费,质量好的则付费.这里只收集免费的代理IP.目前爬取了goubanjia,kuaidaili,66IP,xicidaili这几个网站的代理IP.每个网站IP只爬取了前面10页,后面的基本上不可用.
