JDBC的顶层架构

JDBC的主要目的是使应用程序通过API读写数据库。但是JDBC不仅仅能够用来读取记录，还可以读取数据库的元数据。比如说表的元数据、视图、存储过程等信息，这些通常可以使用DatabaseMetaData类来获取。

根据SUN的设计，JDBC API主要分为几部分：

  • 供应用程序开发者使用的API
  • 供驱动开发者使用的底层API

JDBC的详细架构

JDBC的大部分工作都是DriverManager来完成的，DriverManager是个工厂类，它是唯一能够创建数据库连接的类。DriverManager通过Driver来创建连接。主要流程：

  • 用户代码调用JDBC API
  • JDBC通过Class.forName()加载驱动，比如说Class.forName(“oracle.jdbc.driver.OracleDriver”)
  • Class.forName() 自动创建Driver的实例并在DriverManager中注册
  • 由Driver来跟特定的数据库对话

JDBC 应用程序API的组成

java.sql

  • DriverManager: 加载驱动，创建连接
  • Connection: 代表数据库连接，通过它可以创建Statement, PreparedStatement, CallableStatement 对象.
  • Statement,PreparedStatement , CallableStatement
  • ResultSet: SELECT 语句的结果集
  • SQLException: 异常和错误

javax.sql

  • DataSource
  • Connection Pool
  • Distributed transactions
  • Row Sets
